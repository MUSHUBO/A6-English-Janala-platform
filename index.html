FrequentlyAsked Questions

1. the difference between var, let, and const?
1.1, var: Function-scoped or globally scoped - can be re-declared and updated.
1.2, let: Block-scoped - can be updated but not re-declared in the same scope.
1.3, const: Block-scoped - cannot be updated or re-declared, and must be initialized at declaration.
2. the difference between map(), forEach(), and filter()?
2.1, map(): Creates a new array with the results of applying a function to each element of the original array. It returns a new array.
2.2, forEach(): Executes a provided function for each element in the array but does not return anything (undefined).
2.3, filter(): Creates a new array with all elements that pass a test implemented by a provided function. It returns a new array with the filtered values..
3. explain arrow functions and how they are different from regular functions?
3.1, Arrow functions: Short syntax, no own this, cannot be used as constructors, no arguments.
3.2, Regular functions: Longer syntax, own this, can be used as constructors, has arguments.
4. how JavaScript Promises work?
4.1, A JavaScript Promise is an object representing the eventual completion (or failure) of an asynchronous operation.
- Pending: The promise is still in progress.
- Resolved (Fulfilled): The operation was successful.
- Rejected: The operation failed.
5. how closures work in JavaScript?
5.1, A closure in JavaScript is a function that "remembers" its lexical scope, even when the function is executed outside that scope.
- It gives access to variables from the outer function, even after the outer function has finished executing.
- Closures are useful for data encapsulation and maintaining state..
